INTRODUCCION AL DISEÑO DE SOFTWARE



1.- ¿Que es software?

El software es un término informático que hace referencia a un programa o conjunto de programas de cómputo, así como datos, procedimientos y pautas que permiten realizar distintas tareas en un sistema informático. Comúnmente, se utiliza este término para referirse de una forma muy genérica a los programas de un dispositivo informático. Sin embargo, el software abarca todo aquello que es intangible en un sistema computacional. Aquí tienes algunos detalles adicionales sobre el software:

Tipos

  • Software de sistemas
  • Software de programacion
  • Software de aplicacion
  • Software malicioso
  • Software libre o de propietario

2.- ¿Qué es el modelado de Software?

El modelado de software es una técnica que se utiliza para visualizar y especificar la estructura y el comportamiento de un sistema de información antes de escribir el código. En otras palabras, se crean un conjunto de modelos o “planos del software” que permiten describir aspectos clave del sistema, como los requisitos, la arquitectura y la interacción entre componentes. Estos modelos ayudan a razonar sobre el sistema, favorecen la comunicación, permiten documentar las decisiones y, en algunos casos, incluso permiten la generación automática de código. El Lenguaje Unificado de Modelado (UML) es el estándar más utilizado para el modelado de software. UML permite visualizar, describir y documentar el desarrollo de sistemas de software desde una perspectiva orientada a objetos. A través de diferentes tipos de diagramas, como diagramas de clases, diagramas de casos de uso y diagramas de secuencia, los ingenieros de software pueden representar la estructura y el comportamiento del sistema de manera más clara y comprensible. En resumen, el modelado de software es esencial para planificar y diseñar sistemas antes de escribir el código, y UML proporciona un lenguaje común para expresar estos modelos de manera efectiva. Si tienes más preguntas, no dudes en preguntar.

ejemplos de diseño:

  1. diagramas de clase de gestión de biblioteca, modelar las clases “Libro”, “Usuario” y “Préstamo”.
  2. de casos de uso: sistema de reservas de vuelos;“Buscar vuelos”, “Reservar asiento” y “Cancelar reserva”.
  3. Diagramas de Secuencia: en una aplicación de chat, podríamos modelar la secuencia de mensajes entre usuarios
  4. Diagramas de Despliegue: un sistema de comercio electrónico, podríamos modelar cómo se distribuyen los servidores web y las bases de datos.

3.- ¿Qué es un modelo en el desarrollo de software?

Un modelo de desarrollo de software es una abstracción del proceso de creación de programas informáticos. Estos modelos especifican los pasos y el orden en que se llevan a cabo las actividades durante el desarrollo del software. Imagina un modelo como una representación visual o conceptual del flujo de trabajo y la secuencia en la que se realizan las tareas. Cada modelo tiene sus propias características y enfoques, y la elección del modelo adecuado depende de factores como los requisitos del proyecto, el tamaño, la complejidad y la participación del cliente. Algunos ejemplos de modelos incluyen el modelo en cascada, el modelo iterativo e incremental, el modelo en espiral y el modelo ágil.

4.- ¿Para que se modelar un software?

El modelado del software es una técnica utilizada en ingeniería de software para tratar con la complejidad inherente a los sistemas informáticos. Consiste en crear un conjunto de modelos o “planos del software” que permiten especificar aspectos clave del sistema antes de escribir el código. Estos modelos ayudan al ingeniero de software a visualizar cómo será el sistema a construir, y también pueden utilizarse para la comunicación con el cliente. En resumen, el modelado es el análisis y diseño previo al desarrollo del software, donde se representan las características, requisitos, estructura y comportamiento del sistema mediante modelos

5.- ¿Cuáles son las fases del modelado de Software?

El modelado de software implica varias fases clave que son esenciales para el desarrollo exitoso de aplicaciones y sistemas. A continuación, describiré estas fases en detalle:

  1. Definicion de requisitos
    • Adopta un enfoque centrado en el usuario y realiza investigaciones exhaustivas.
  2. Analizis y diseño
    • El objetivo es crear un diseño sólido que guíe la implementación
  3. Implementacion
    • Los desarrolladores escriben el software siguiendo las especificaciones y las pautas establecidas.
  4. Pruebas
    • La calidad y la funcionalidad del software se verifican en esta etapa.
  5. Despliegue
      6. Se instala y configura en el entorno operativo para que los usuarios lo utilicen.
  6. Mantenimiento

6.- ¿Qué es arquitectura de software?

La arquitectura de software se refiere a la estructura y diseño de un sistema de software. Es una representación de alto nivel que define cómo los componentes del software interactúan entre sí, cómo se organizan y cómo cumplen con los requisitos funcionales y no funcionales del sistema. En otras palabras, es la planificación teórica previa a la implementación, donde se establecen patrones, abstracciones y modelos para guiar el desarrollo de una aplicación o servicio. La arquitectura nos permite tomar decisiones fundamentales antes de escribir código, considerando aspectos como el costo, el tiempo de desarrollo y el número de usuarios que soportará el sistema.

7.- ¿Qué es la interfaz de usuario?

La interfaz de usuario (UI) es el punto de interacción entre un usuario y un dispositivo digital o sistema. En otras palabras, es el medio a través del cual una persona controla una aplicación de software o un dispositivo de hardware. Aquí tienes algunas características clave de la interfaz de usuario:

  1. Atractivo visual: El diseño debe ser agradable a la vista, permitiendo que el usuario se sienta cómodo al interactuar con el programa.

  2. Claridad: La información transmitida al usuario debe ser clara y concisa para evitar errores o confusiones durante la interacción con el software.

  3. Coherencia: Todos los elementos de una aplicación deben mantener unidad en su diseño y propósito, lo que permite a los usuarios crear patrones de uso de forma intuitiva.

8.- ¿Cuáles son los tipos de modelado de Software?

Los modelos de desarrollo de software son abstracciones del proceso de creación de programas y aplicaciones. Estos modelos especifican las etapas y el orden en que se llevan a cabo las actividades durante el desarrollo. Aquí tienes algunos de los tipos más comunes:

  • Modelo en cascada: Este es un enfoque secuencial y lineal donde cada fase (requisitos, diseño, implementación, pruebas, mantenimiento) se completa antes de pasar a la siguiente. Es adecuado para proyectos con requisitos estables y bien definidos
  • Modelo iterativo: En este enfoque, se realizan iteraciones o ciclos repetidos de desarrollo. Cada iteración agrega funcionalidad y refina el producto. Es útil cuando los requisitos pueden cambiar o evolucionar durante el proceso
  • Modelo incremental: Se construye el sistema en pequeñas partes o módulos, y cada incremento se agrega al sistema existente. Es ideal para proyectos grandes y complejos
  • Modelo en espiral: Combina elementos del modelo en cascada y el enfoque iterativo. Se evalúan riesgos y se toman decisiones basadas en ellos. Es útil para proyectos con incertidumbre y riesgos
  • Modelo ágil: Incluye metodologías como Scrum, Kanban y Programación Extrema (XP). Se enfoca en la colaboración, la adaptabilidad y la entrega continua de incrementos funcionales

9.- ¿Qué es UML?

El Lenguaje Unificado de Modelado (UML) es un lenguaje de modelado estandarizado que permite a los desarrolladores especificar, visualizar, construir y documentar artefactos de un sistema de software. UML es un aspecto importante involucrado en el desarrollo de software orientado a objetos. En términos más sencillos, UML es un lenguaje visual en el que se trabaja con cajas, flechas y diagramas para representar las ideas sobre cómo estructurar programas de una forma más visual. Aunque no es un lenguaje de programación en sí mismo, se utiliza para describir los límites, la estructura y el comportamiento de sistemas y objetos en el mundo del desarrollo de software.

10.- ¿Qué es un programador de software?

Un programador es aquella persona que elabora programas de computadora. Para la creación de software, los y las programadores/as utilizan lenguajes de programación, que son similares a idiomas que permiten entenderse con el ordenador y escribir instrucciones que generan nuevos programas.
En resumen, un programador escribe, depura y mantiene el código fuente de un programa informático, que ejecuta el hardware de una computadora, para realizar una tarea determinada.
A veces también se les denomina desarrolladores de software, aunque estrictamente forman parte de un equipo de personas de distintas especialidades, mayormente informáticas. La programación es una de las principales disciplinas dentro de la informática.

11.- ¿Qué es un programador de software?

Un diseñador de software es un profesional especializado en la creación y diseño de programas informáticos.
Su principal objetivo es transformar las necesidades y requerimientos de los usuarios en soluciones tecnológicas funcionales y eficientes.
El trabajo de un diseñador de software implica comprender la necesidad, desarrollar una solución y emplear la metodología de manera accesible y lista para el usuario.
Para lograrlo, el diseñador debe estar bien versado en programación informática general y utilizar conceptos avanzados y rudimentarios. Algunas de las consideraciones clave en el diseño de software incluyen la compatibilidad, el mantenimiento, la confiabilidad, la seguridad y la reutilización.
En resumen, los diseñadores de software son esenciales para cualquier equipo informático, ya que crean sistemas y programas que mejoran la experiencia de los usuarios y resuelven problemas tecnológicos.

12.- ¿Qué es un programador de software?

Un algoritmo es un conjunto ordenado y finito de operaciones simples que nos permite hallar la solución a un problema. Estas operaciones se ejecutan siguiendo una serie de instrucciones definidas, ordenadas y finitas. Algunas características clave de los algoritmos son:

  1. Inicio y fin
  2. Secuencia
  3. Abstracción
  4. Finitud

Ejemplos

  • Recetas de cocina
  • Manuales
  • Operaciones matemáticas
  • Informática

13.- ¿Que es la ingeniería de requerimientos?

La ingeniería de requerimientos es el proceso de recopilación, análisis y documentación de los requisitos de un sistema de software. Es una parte fundamental del proceso de desarrollo de software, ya que garantiza que el producto final satisfaga las necesidades del cliente. En otras palabras, se trata de entender y definir las necesidades del cliente y traducirlas en especificaciones claras y precisas para el sistema que se va a construir. Este proceso implica la colaboración entre el cliente, los usuarios y los diseñadores del software para establecer los objetivos, funciones y restricciones del sistema. A través de la ingeniería de requerimientos, se crea una base sólida para acuerdos comunes entre todas las partes involucradas y se describen las funciones que realizará el sistema.
